# ทางเลือกการ Deploy (Deployment Options)

ถูกต้องครับ **Web Hosting ทั่วไป (ที่รัน PHP)** จะ **ไม่รองรับ** โปรเจกต์นี้ครับ เพราะต้องใช้ Node.js

ส่วน **Cloudflare** (Cloudflare Pages) นั้นดีมากและฟรี แต่มีข้อจำกัดคือ **ไม่รองรับฐานข้อมูลแบบไฟล์ (SQLite)** ที่เราใช้อยู่ครับ (เพราะ Cloudflare เป็นแบบ Serverless ไม่มีฮาร์ดดิสก์ให้เก็บไฟล์ `dev.db`)

ดังนั้น ผมขอแนะนำ 2 ทางเลือกที่ดีที่สุดครับ:

## ทางเลือกที่ 1: ใช้ Vercel (แนะนำ! ง่ายและฟรี)
Vercel คือผู้สร้าง Next.js เหมาะที่สุดสำหรับการ Deploy ครับ
**แต่** เราต้องเปลี่ยนฐานข้อมูลจาก SQLite (ไฟล์) ไปเป็น **Postgres (Cloud)** ซึ่ง Vercel มีให้ใช้ฟรี (Vercel Postgres)

**ขั้นตอน:**
1. สมัครสมาชิก [Vercel.com](https://vercel.com) (ใช้ GitHub Login ได้เลย)
2. ติดตั้ง Vercel CLI ในเครื่อง: `npm i -g vercel`
3. เปลี่ยนการตั้งค่า Database ในโปรเจกต์ (ผมทำให้ได้ครับ)
4. รันคำสั่ง `vercel` เพื่อ Deploy

**ข้อดี**: ฟรี, เร็ว, รองรับ Next.js สมบูรณ์แบบ
**ข้อเสีย**: ต้องเปลี่ยน Database (แต่ผมจัดการแก้โค้ดให้ได้)

---

## ทางเลือกที่ 2: ใช้ VPS (ถ้าอยากใช้ SQLite เหมือนเดิม)
เช่า VPS (เช่น DigitalOcean, Ruk-Com Cloud, หรือ Cloud ไทยอื่นๆ) ราคาประมาณ 150-300 บาท/เดือน
**ข้อดี**: ไม่ต้องแก้โค้ด Database, ข้อมูลเก็บเป็นไฟล์ `dev.db` เหมือนในเครื่อง
**ข้อเสีย**: ต้องตั้งค่า Server เอง (ลง Node.js, ตั้งค่า Domain, SSL) ซึ่งยุ่งยากกว่ามาก

---

## สรุป
ผมแนะนำให้ไป **ทางเลือกที่ 1 (Vercel)** ครับ เพราะ:
1. **ฟรี** ตลอดชีพสำหรับโปรเจกต์ส่วนตัว
2. **ไม่ต้องดูแล Server** เอง
3. **ผมสามารถแก้โค้ดให้รองรับ Vercel Postgres ได้เลยตอนนี้**

คุณต้องการให้ผมปรับแก้โค้ดเพื่อเตรียม Deploy ขึ้น **Vercel** เลยไหมครับ? (จะเปลี่ยนจาก SQLite -> Postgres)
